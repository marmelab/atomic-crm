---
name: No push proposals
description: Non proporre mai git push — l'utente decide quando pushare
type: feedback
---

Non chiedere mai "vuoi che faccia push?" o proporre push dopo un commit.
L'utente lo chiederà esplicitamente quando sarà il momento.

**Why:** L'obiettivo delle sessioni è bonificare il progetto, non rilasciare. Il push è una decisione dell'utente, non dell'agente.

**How to apply:** Dopo un commit, procedere col prossimo task o riepilogare. Mai menzionare push.
