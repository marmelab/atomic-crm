title: Agent Harness

Atomic CRM ships a local **agent harness** for Claude Code: a coordinated team of agents that turns a feature request in natural language into working code. Instead of implementing a change yourself, you can describe what you want and the harness plans, implements, reviews, and merges it through a deterministic pipeline.

## Prerequisites

- The [Claude Code CLI](https://docs.claude.com/en/docs/claude-code) installed and authenticated (an Anthropic API key or a Claude subscription).
- Project dependencies installed (`make install`).
- For full-stack runs only: a local Supabase instance up (`make start-supabase`).

## Launching a Session

To start a Claude Code session directly with the harness:

```sh
make harness
```

This launches the **chat-orchestrator** agent in automatic-permission mode. Type your request once the session opens, and the orchestrator takes over.

You can also pre-fill the first message:

```sh
make harness REQUEST="Add a reason for a lost deal"
```

Or, you can start a plan mode and at the end, **chat-orchestrator** will implement it with the harness.

:::tip
**Heads up**: The harness runs in automatic-permission mode, which means it can execute arbitrary commands on your machine. We advise you to run the harness in a disposable environment, such as a DevContainer (already configured in the repo) or a throwaway VM.
:::

The orchestrator will determine the type of your request and launch the appropriate sub-agents. There are two types of requests:

- **Simple**: a single cosmetic edit, a single-field change on an existing entity, or a filter reusing existing components. One developer agent, no planning overhead.
- **Complex**: everything else. The planner decomposes it into tickets; developers, reviewers, and the merger run as a coordinated wave.

Then, all the work will take place in a git worktree to keep your branch clean. The orchestrator creates a new branch and, once the change is complete, **merges it automatically into the branch you started from**.

## Undoing a Session

Because the merge is automatic, if you need to roll back, you can use the command:

```sh
make clean-harness
```

This removes everything the last session created (worktrees, session branches) and hard-resets your start branch to the exact commit it was on before the session began.

:::caution
`make clean-harness` is a hard reset: any uncommitted work is stashed first (recover it with `git stash pop`), but committed work made *after* the session started on that branch is also dropped. Run it only when you want to discard the whole session.
:::

## Disabling the harness

If you'd rather have the change implemented directly, without the harness, say so in your request. Any of these opt out:

- `#no-harness`
- *"implement directly"*
- *"without the agent team"*
- *"skip harness"*

The main thread then implements the change itself, no agents, even after a plan is approved. Add *"no-harness for this session"* to keep it off for the whole session.

## Monitoring a Session

The harness writes a detailed trace of every agent, hook, and ticket. Three commands let you follow it:

```sh
make watch                  # live monitor of the most recent session
make monitor                # one-shot summary of the most recent session
make monitor SESSION=<id>   # summary of a specific session
make sessions               # list known sessions, newest first
```

`make watch` re-renders a live view of the active agents, hook activity, and ticket statuses, useful to keep open in a second terminal while a COMPLEX session runs. `make monitor` prints the same information as a single snapshot.

## CRM Builder: A Visual, Ready-Made Environment

[**CRM Builder**](https://github.com/marmelab/crm-builder) wraps the same harness in a containerized, browser-based environment, with no local setup. It adds:

- A containerized environment so you can run the harness without security concerns.
- A pre-installed toolchain, no need to set up Claude Code, Node, or Supabase yourself.
- A live preview of the CRM, updated as the harness applies your changes.
- A session dashboard to follow each run's agents and progress.
- Demo and full-stack mode switching from the UI.
